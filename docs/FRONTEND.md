# Frontend Architecture

## Overview

Kasku has no JavaScript build step and no npm. All HTML is generated by the Go server using **Templ** (type-safe Go HTML templates). Client-side interactivity comes from **HTMX**, loaded from a CDN. Styling uses **Tailwind CSS**, also from CDN.

This makes the frontend a zero-dependency part of the Go monolith.

## Technology Choices

| Tool | Why |
|------|-----|
| **Templ** | Type-safe HTML in Go; compile-time errors for template issues; no reflection |
| **HTMX** | HTML-over-the-wire; dynamic updates without writing JavaScript |
| **Tailwind CSS** | Utility-first; no custom CSS file needed for most UI |

## Template Layout

```
web/templates/
├── layout/
│   ├── base.templ        # Full-page shell (Tailwind CDN, HTMX CDN, navbar, modal div)
│   └── auth.templ        # Minimal centered shell for login/register pages
├── components/           # Reusable fragments
│   ├── helpers.go        # FormatMoney, FormatMonth, TodayISO (plain Go, no templ)
│   ├── flash.templ       # Success/error flash message bar
│   ├── modal.templ       # Modal overlay shell (HTMX fills the content)
│   ├── navbar.templ      # Top navigation bar
│   ├── summary_card.templ# Single metric card (income / expenses / net)
│   ├── wallet_card.templ # Wallet display card with edit/deactivate buttons
│   ├── ledger_row.templ  # Single table row in the ledger
│   └── month_switcher.templ # Prev/next month navigation
├── pages/                # Full page templates (one per route)
│   ├── login.templ
│   ├── register.templ
│   ├── invite.templ
│   ├── dashboard.templ
│   ├── wallets.templ
│   ├── wallet_form.templ # Wallet create/edit form (returned as modal content)
│   ├── ledger.templ
│   ├── ledger_form.templ # Add-entry form (returned as modal content)
│   └── family.templ
└── partials/             # HTMX swap targets (HTML fragments, no layout)
    ├── ledger_table.templ   # <div id="ledger-table-body"> with table rows
    └── monthly_totals.templ # Summary cards with hx-swap-oob
```

## Templ Basics

### Defining a component

```go
// web/templates/components/flash.templ
templ Flash(f FlashData) {
    <div class="...">{ f.Message }</div>
}
```

### Composing components

```go
// Pass children using the layout composition pattern
templ Base(title string, user *model.User, flash FlashData) {
    <!DOCTYPE html>
    ...
    @navbar(user)
    { children... }   // <-- children are injected here
}

// Page that uses Base:
templ DashboardPage(d DashboardData) {
    @layout.Base("Dashboard", d.User, d.Flash) {
        <h1>Hello</h1>   // this becomes children
    }
}
```

### Calling a Go function inside a template

```go
<p>{ components.FormatMoney(wallet.Balance) }</p>
```

### Conditional rendering

```go
if user.IsAdmin() {
    @AdminPanel()
}
```

### Looping

```go
for _, w := range wallets {
    @WalletCard(w)
}
```

## HTMX Patterns

### Modal open (fetch modal content from server)

```html
<button
  hx-get="/wallets/new"
  hx-target="#modal"
  hx-swap="innerHTML">
  New Wallet
</button>
```

The server returns a `ModalShell` component containing the form fragment. HTMX injects it into the empty `<div id="modal">` in the base layout.

### Modal close

Two mechanisms:
1. `HX-Trigger: closeModal` response header — the Go handler sets this after a successful form submission. A `<script>` in `base.templ` listens for the event and clears `#modal`.
2. Clicking the backdrop or the ✕ button — inline `hx-on:click` that clears `#modal` directly.

```html
<!-- In base.templ -->
<script>
  document.body.addEventListener('closeModal', function() {
    document.getElementById('modal').innerHTML = '';
  });
</script>
```

### Form submission with HTMX

```html
<form
  hx-post="/ledger"
  hx-target="#ledger-content"
  hx-swap="innerHTML">
  ...
</form>
```

The server responds with a fresh `LedgerTable` partial. HTMX replaces the content of `#ledger-content`.

On validation errors, the server returns HTTP 422 with the form fragment (including error messages). HTMX swaps the error form into the modal, keeping it open.

### Month navigation (filter without page reload)

```html
<button
  hx-get="/ledger/entries?month=2025-01&wallet=0"
  hx-target="#ledger-content"
  hx-swap="innerHTML">
  ‹
</button>
```

`GET /ledger/entries` returns a `LedgerTable` partial. The month switcher component builds the correct query string from Go functions.

### Wallet delete (remove card from DOM)

```html
<button
  hx-delete="/wallets/42"
  hx-target="#wallet-42-card"
  hx-swap="outerHTML"
  hx-confirm="Deactivate this wallet?">
  Deactivate
</button>
```

The server soft-deletes the wallet and returns an empty 200 response. HTMX replaces the card element with nothing, removing it from the page.

## Styling Guide

Tailwind classes are applied directly in templates. A few conventions:

- **Cards**: `bg-white rounded-xl border border-gray-200 shadow-sm p-5`
- **Primary button**: `bg-indigo-600 hover:bg-indigo-700 text-white font-medium py-2.5 rounded-lg text-sm transition-colors`
- **Secondary button**: `bg-gray-100 hover:bg-gray-200 text-gray-700 font-medium py-2.5 rounded-lg text-sm`
- **Form input**: `w-full rounded-lg border border-gray-300 px-3 py-2 text-sm focus:outline-none focus:ring-2 focus:ring-indigo-500`
- **Success text**: `text-green-600`
- **Danger text**: `text-red-500`

## Adding a New Page

1. Create `web/templates/pages/mypage.templ` with a `MyPage(data MyData)` component that wraps `@layout.Base(...)`
2. Create a handler method that calls `handler.Render(w, r, 200, pages.MyPage(data))`
3. Add the route to `internal/router/router.go`
4. Run `templ generate` (or let `make templ-watch` pick it up)

## Adding a New HTMX Partial

1. Create `web/templates/partials/myfragment.templ`
2. Give the root element a stable `id` attribute (e.g., `id="my-fragment"`)
3. Return it from a handler: `handler.Render(w, r, 200, partials.MyFragment(data))`
4. Point your HTMX button at the route with `hx-target="#my-fragment"` and `hx-swap="innerHTML"`

## CDN Versions

These are pinned in `web/templates/layout/base.templ` and `auth.templ`:

| Library | Version | URL |
|---------|---------|-----|
| Tailwind CSS | latest (play CDN) | `https://cdn.tailwindcss.com` |
| HTMX | 2.0.4 | `https://unpkg.com/htmx.org@2.0.4/dist/htmx.min.js` |

For production, consider self-hosting these files or using a pinned CDN URL with SRI hash.

## Development Workflow

```bash
# Watch templ files and regenerate on save
make templ-watch

# In a second terminal, hot-reload the Go binary
make dev
```

After saving any `.templ` file, `templ` regenerates the `_templ.go` file. `air` detects the changed `.go` file and rebuilds+restarts the server automatically. Refresh the browser to see changes.
